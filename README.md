1. 引言
1.1 编写目的
本文档详细描述移动应用“Soker-MovingHakerTools”的开发需求、功能设计、运行环境和技术约束，为后续开发、测试和维护提供指导。该应用定位为轻量级移动工具集合，所有功能均不依赖开发者自建服务器和数据库（无后端数据存储），但部分功能需联网调用第三方API或进行网络请求，以实现高级特性。
1.2 项目背景
项目名称：Soker-MovingHakerTools
开发者：吴永贤
该应用针对移动端用户，提供一系列实用工具，强调轻量、集成化和高效操作。所有用户数据均存储在本地设备，不涉及云端同步或服务器存储，最大程度保护隐私。
1.3 术语说明

Termux：Android终端模拟器和Linux环境应用，支持运行Python等脚本。
悬浮窗：Android系统允许的应用窗口悬浮显示功能。
第三方API：指公开的云服务API（如身份证识别API、AI模型API等），需联网调用。
无服务器依赖：不依赖开发者自建后端服务器和数据库。

Android官方开发文档
Termux开源项目文档
相关第三方API文档（身份证识别、AI接口等）

2. 项目概述
2.1 目标
开发一款轻量、实用的移动工具应用，所有功能不依赖自建服务器和数据库，仅通过本地处理或调用第三方公开API实现。支持部分功能离线使用（如音乐播放、笔记、文件预览），部分功能需联网（如身份证识别、AI接口、高级网页请求）。
2.2 用户群体

技术爱好者、开发者
需要集成化移动工具的用户
Android设备用户（优先支持Android 10+）

2.3 运行环境

平台：Android（最低版本Android 8.0，推荐Android 12+）
权限要求：相机（身份证识别）、存储读写、悬浮窗、网络（多项功能必需）、局域网访问等
无自建服务器和数据库依赖
应用大小控制在50MB以内，确保轻量

2.4 约束与限制

所有用户数据均本地存储，不上传至自建服务器
需调用第三方API的功能必须明确提示联网和隐私政策
需遵守Android权限管理规范和第三方API使用协议

3. 功能需求
3.1 总体功能列表
应用主要功能模块如下（身份证识别已独立为单独模块）：
模块序号,功能模块,主要描述,是否需联网
1,身份证识别,独立功能模块，使用相机拍摄身份证，调用第三方云API进行高精度识别，返回结构化信息。,是（调用云API）
2,身份证识别登录,基于身份证识别模块结果进行本地身份验证，作为应用主要登录方式。,首次需联网
3,主页欢迎,登录成功后显示欢迎页面，展示功能入口、快捷工具栏和个性化问候。,否
4,接入Termux Python运行环境,集成Termux API，支持在应用内运行Python脚本，提供代码编辑器和终端模拟。,否（脚本本地运行）
5,高级网页请求和预览功能,支持自定义Headers、代理、JS注入的发包工具，网页预览和保存功能。,是（网络请求）
6,局域网文件传输（共享编辑）,通过Wi-Fi局域网实现设备间文件传输，支持实时共享编辑。,是（局域网）
7,音乐播放功能,本地音乐文件播放器，支持播放列表、后台播放和简单均衡器。,否
8,AI接口,调用第三方AI云API（如通义千问、DeepSeek、Groq等），提供聊天、文本生成等功能。,是（调用云API）
9,笔记功能,支持Markdown、DOCX、XLSX格式的笔记创建、编辑和本地保存。,否
10,悬浮窗功能,支持关键工具（如终端、笔记、AI聊天）以悬浮窗形式显示，便于多任务操作。,视模块而定
11,多文件存储和预览功能,本地文件管理器，支持多种格式预览（PDF、图像、视频、办公文档等）。,否
3.2 详细功能描述
3.2.1 身份证识别（独立模块）

使用设备相机实时拍摄或从相册选择身份证照片
调用第三方云端身份证识别API（需配置API Key，用户需同意联网上传）
返回结构化数据：姓名、身份证号、地址、有效期等
提供识别结果预览与手动修正功能
本地缓存最近识别结果（仅存储哈希或必要信息）

3.2.2 身份证识别登录

调用身份证识别模块获取身份信息
首次识别后本地存储身份哈希，用于后续快速验证
支持手动输入或备用登录方式

3.2.3 主页欢迎

显示用户姓名（从身份证识别提取）
网格或列表形式展示所有功能入口
支持自定义主题、快捷方式和最近使用记录

3.2.4 接入Termux Python运行环境

需用户安装Termux并授予存储权限
应用内嵌入终端视图，支持Python脚本编辑、运行和输出显示

3.2.5 高级网页请求和预览功能

高级发包工具：支持GET/POST/PUT等方法，自定义Headers、Cookies、代理
支持JS注入、响应拦截
内置浏览器预览，支持页面截图和HTML本地保存

3.2.6 局域网文件传输（共享编辑）

自动扫描局域网内同应用设备或创建热点
支持文件发送/接收
实时协作编辑文本文件（基于WebSocket局域网通信）

3.2.7 音乐播放功能

扫描本地音乐文件构建播放列表
支持后台播放、通知栏控制、简单均衡器

3.2.8 AI接口

支持配置多个第三方AI API（如通义千问、Claude、Gemini、Groq等）
提供聊天界面、代码生成、文本处理等功能
支持历史记录本地保存

3.2.9 笔记功能

Markdown编辑器（实时预览）
支持DOCX、XLSX导入/导出（使用本地库处理）
本地分类管理与搜索

3.2.10 悬浮窗功能

支持终端、AI聊天、笔记、网页预览等模块悬浮显示
可拖拽、缩放、透明度调节

3.2.11 多文件存储和预览功能

本地文件浏览器
支持图像、PDF、视频、办公文档等多格式预览

4. 非功能需求
4.1 性能需求

启动时间 < 3秒
本地功能响应时间 < 1秒
联网功能响应时间视网络情况（目标 < 5秒）

4.2 安全性需求

所有需上传数据（如身份证照片、AI请求）前必须弹出明确隐私同意弹窗
本地存储数据加密
支持配置API Key本地保存（加密）
权限最小化原则

4.3 可用性需求

简洁UI，Material Design风格
支持暗黑模式
多语言（中文优先）
联网功能提供清晰提示和离线降级

4.4 可靠性需求

网络异常时优雅处理（如缓存、提示重试）
完整异常捕获与崩溃报告（可选本地日志）

5. 外部接口需求
5.1 用户界面

主界面：底部导航栏 + 侧边抽屉
身份证识别与AI接口等联网模块提供独立入口

5.2 软件接口

Termux API
第三方云API（身份证识别、AI模型等，需集成相应SDK或HTTP调用）
Android系统API（相机、存储、悬浮窗、网络）
